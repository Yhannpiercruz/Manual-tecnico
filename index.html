Operación técnica

Levantamiento con Docker, ejecución local, diagnóstico y mantenimiento

Levantar todo con Docker

  • Requisitos: Docker Desktop y docker-compose.
  • Desde la raíz del proyecto: docker-compose up -d
  • Verificación: docker ps y URLs en URLS-DEL-SISTEMA.md.

Backend (Gradle/Java)

  • JDK recomendado: 21 (ver build_log_jdk21.txt).
  • Compilar: gradlew clean build -x test
  • Ejecutar: start-server.bat o gradlew :renacer-institucional:bootRun
  • Logs: revisar build_log_*.txt y build_error_stacktrace.txt en la raíz.

Frontend

  • Vite/React: cd Frontend, npm install, npm run dev.
  • Vistas estáticas: abrir Frontend/index.html o servir con Nginx (ver nginx.conf).
  • Variables: ajustar .env.production y vite.config.js si cambia el host del backend.

Diagnóstico

  • Build backend: build_log_*.txt, build_output.txt.
  • Scripts útiles: limpiar-gradle.bat, resetear-base-datos.bat, clean_usecases.ps1.
  • Errores comunes: versión JDK incorrecta, dependencias Gradle, endpoints en Frontend/controller/js/api.js.

Configuración

  • Backend: gradle.properties, perfiles y variables según entorno.
  • Frontend: .env.production, vite.config.js.
  • Nginx/Docker: revisar Frontend/nginx.conf y Dockerfiles en raíz y módulos.

Despliegue

  • Railway: seguir RAILWAY_COPIAR_PEGAR.md y Resumen_Cambios_Railway.md.
  • Variables/secretos: definir equivalentes a .env.production en el proveedor.

Mantenimiento

  • Base de datos de desarrollo: Backend/renacer-institucional/data (renacer.mv.db).
  • Reset: resetear-base-datos.bat.
  • Limpieza Gradle: limpiar-gradle.bat.

Referencias rápidas

  • Guías: INSTRUCCIONES-RECOMPILAR.md, INICIO-FRONTEND.md, README-REACT.md.
  • Scripts: iniciar-todo.bat, iniciar-todo-cmd.bat, iniciar-todo-ps1.ps1.
